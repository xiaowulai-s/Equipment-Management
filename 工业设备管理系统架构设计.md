# 工业设备管理系统 - 架构设计文档

## 一、架构设计理念

### 1.1 核心设计思想

本系统采用**配置驱动 + 分层架构 + 插件化 + 多设备调度**的综合设计理念，实现代码逻辑通用化，产品差异配置化的目标。

**设计核心原则：**
- ✅ **高度模块化**：各功能模块松耦合，便于维护和扩展
- ✅ **配置驱动**：产品差异通过配置文件实现，无需修改核心代码
- ✅ **实时响应**：多线程/异步处理确保界面流畅和数据实时性
- ✅ **工业级可靠性**：完善的错误处理、重连机制和日志系统
- ✅ **易于扩展**：支持新增设备类型和通信协议

## 二、系统整体架构

系统采用**五层架构模型**，各层职责明确，接口清晰，实现了业务逻辑与技术细节的分离。

```
┌──────────────────────────────┐
│        用户界面层 (UI)         │
│  PyQt6 / PySide6              │
└──────────────┬───────────────┘
               │
┌──────────────▼───────────────┐
│       业务逻辑控制层          │
│  设备管理 / 数据处理 / 命令调度  │
└──────────────┬───────────────┘
               │
┌──────────────▼───────────────┐
│       设备模型与配置层         │
│  产品Profile / 寄存器映射      │
└──────────────┬───────────────┘
               │
┌──────────────▼───────────────┐
│       协议与通信驱动层         │
│  Modbus TCP / 分包合并逻辑     │
└──────────────┬───────────────┘
               │
┌──────────────▼───────────────┐
│     数据持久化与日志层         │
│  SQLite / 通信日志 / 历史记录   │
└──────────────────────────────┘
```

## 三、分层架构详细设计

### 3.1 用户界面层 (UI Layer)

**技术选型：** PyQt6 或 PySide6（推荐PyQt6，社区活跃，文档丰富）

**核心设计：**
- **动态渲染**：界面元素根据选中的产品型号，从配置文件中读取变量列表，动态生成数据卡片、仪表盘或控制按钮
- **多窗口/多标签管理**：支持同时打开多个设备的监控界面
- **实时数据展示**：表格、实时曲线、仪表盘等多种数据可视化方式
- **工业级布局**：
  ```
  ┌────────────────────────────────────────────┐
  │ 顶部：设备总览（在线状态）                 │
  ├──────────────┬─────────────────────────────┤
  │ 左侧设备列表 │ 右侧数据监控区              │
  │              │                             │
  │              │  表格 + 实时数据            │
  │              │                             │
  ├──────────────┴─────────────────────────────┤
  │ 底部：日志区 / 通信记录                    │
  └────────────────────────────────────────────┘
  ```

**界面风格：** 现代工业风（Fluent Design），支持深色主题，专业的工业物联网美学设计

### 3.2 业务逻辑控制层 (Controller Layer)

**核心职责：**
- **设备生命周期管理**：负责设备的连接、断开、重连逻辑
- **任务调度器**：管理高频读取（如实时压力）与低频读取（如版本号）的优先级，防止Modbus队列阻塞
- **命令处理**：接收UI层的操作指令，转换为设备可执行的命令格式
- **数据处理**：对设备返回的数据进行解析、转换和校验

**设计亮点：**
- **状态管理**：统一管理设备的连接状态、数据状态和操作状态
- **错误处理**：完善的异常捕获和处理机制，确保系统稳定性
- **事件驱动**：采用信号槽机制实现模块间通信

### 3.3 设备模型与配置层 (Device & Config Layer)

**设计思想：** 代码逻辑通用化，产品差异配置化

**核心组件：**
- **产品Profile (JSON/YAML)**：为每款产品编写配置文件，定义变量与寄存器的映射关系
  ```json
  {
    "产品型号": "PumpStation-A",

      {"变量名": "压力", "地址": 0x0002, "类型": "float", "读写权限": "R", "缩放因子": 0.1},
      {"变量名": "运行状态", "地址": 0x0003, "类型": "int", "读写权限": "R"}
    ]
  }
  ```

  ```python
          pass

  ```

- **设备实例化**：软件启动时，根据配置文件实例化对应的设备对象，自动映射寄存器地址

### 3.4 协议与通信驱动层 (Protocol & Comm Layer)

**技术选型：** pymodbus（支持Modbus TCP，异步通信，社区活跃）

**核心设计：**
- **Modbus TCP引擎**：基于pymodbus封装，提供统一的通信接口
  ```python
  from pymodbus.client import ModbusTcpClient
  
  class ModbusClient:
      def __init__(self, ip, port):
          self.client = ModbusTcpClient(ip, port=port)
      
      def read_holding(self, addr, count):
          return self.client.read_holding_registers(addr, count)
      
      def write_register(self, addr, value):
          return self.client.write_register(addr, value)
  ```

- **分包合并逻辑**：自动分析需要读取的寄存器地址，将连续的地址合并为一个Modbus读取指令，大幅提升通信效率
- **连接池管理**：维护设备连接池，避免频繁创建和关闭连接
- **超时与重连机制**：完善的网络异常处理和自动重连策略

### 3.5 数据持久化与日志层 (Data Layer)

**核心组件：**
- **SQLite数据库**：轻量级本地数据库，存储所有设备的历史运行数据
- **日志引擎**：
  - 记录每一帧发送和接收的HEX原文，便于售后排查通讯故障
  - 分级日志系统（调试、信息、警告、错误）
  - 日志轮转机制，避免日志文件过大

**设计亮点：**
- **高效数据存储**：采用批量写入和定时提交策略，提高数据存储效率
- **历史数据查询**：支持按时间范围、设备类型、变量名等条件查询历史数据
- **数据导出**：支持将历史数据导出为Excel或CSV格式

## 四、核心模块设计

### 4.1 多设备调度模块

**设计目标：** 高效管理多个设备的并发通信，确保数据实时性和系统稳定性

**可选方案：**

#### 方案A：多线程 + 队列（简单稳定）
- 为每个设备分配一个独立的读写线程
- 使用队列实现线程间通信
- 适用于设备数量较少（<50）的场景

#### 方案B：asyncio（高并发）
- 使用异步IO机制处理设备通信
- 单线程异步执行，减少线程切换开销
- 适用于设备数量较多（>50）的场景

**调度策略：**
```python
# 伪代码实现
while True:
    for device in devices:
        if device.need_read():
            asyncio.create_task(device.read_data())
    await asyncio.sleep(0.1)
```

### 4.2 命令系统模块

**设计目标：** 统一管理设备控制命令，支持动态指令映射

**核心设计：**

```python
        self.addr = addr  # 寄存器地址
        self.value = value  # 命令值
        self.func_code = func_code  # 功能码（05/06/16等）
        
    def to_modbus_frame(self):
        # 转换为Modbus帧格式
        pass
```

**支持的命令类型：**
- ✅ 启动/停止控制
- ✅ 参数设置
- ✅ 复位操作
- ✅ 批量配置

### 4.3 寄存器映射模块

**设计目标：** 实现变量与寄存器地址的灵活映射，支持不同产品的快速适配

**核心设计：**
```python
DEVICE_MAP = {
    "temperature": {"addr": 0x0001, "type": "float", "scale": 0.1},
    "humidity": {"addr": 0x0002, "type": "float", "scale": 0.1},
    "status": {"addr": 0x0003, "type": "int"}
}
```

**设计亮点：**
- ✅ 支持不同数据类型（int、float、bool等）
- ✅ 支持缩放因子配置
- ✅ 支持读写权限控制
- ✅ 支持位操作（用于状态寄存器）

### 4.4 动态指令映射模块

**设计目标：** 根据配置文件动态生成设备控制指令

**核心功能：**
- 针对不同操作（如"启动"、"复位"、"参数修改"），在配置文件中定义具体的功能码和对应地址
- 界面点击按钮时自动触发对应的帧封装
- 支持批量指令生成和发送

## 七、UI设计

### 7.1 界面设计原则

- ✅ **工业级专业感**：采用深色主题，蓝色/青色为主色调
- ✅ **信息层次清晰**：重要信息突出显示，次要信息适当弱化

- **寄存器表格**：显示设备寄存器的地址、值和状态
- **控制区**：提供设备操作按钮和参数设置界面
- **日志区**：显示实时通信日志和系统日志

## 八、扩展与维护

## 九、总结

本架构设计采用**配置驱动 + 分层架构 + 插件化 + 多设备调度**的综合设计理念，实现了一个高度模块化、可扩展、稳定可靠的工业设备管理系统。

**设计优势：**
- ✅ **高度可扩展**：支持新增设备类型和通信协议
- ✅ **易于维护**：模块化设计，各功能模块松耦合
- ✅ **配置灵活**：产品差异通过配置文件实现，无需修改核心代码
- ✅ **性能优异**：采用多线程/异步处理，支持高并发设备管理
- ✅ **稳定可靠**：完善的错误处理和日志系统，确保系统稳定性

本架构设计适用于各种工业设备管理场景，特别是需要管理多种产品、使用统一通信协议的应用场景，能够帮助企业快速构建稳定可靠的工业设备管理系统。
