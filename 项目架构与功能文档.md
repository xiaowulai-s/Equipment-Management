# 工业设备管理系统 - 项目架构与功能文档

## 1. 项目概述

本项目是一个基于PySide6开发的工业设备管理系统，主要用于管理和监控工业设备，支持Modbus通信协议（TCP/RTU/ASCII）。系统采用分层架构设计，具有良好的扩展性和可维护性。

## 2. 项目架构

### 2.1 总体架构

系统采用典型的分层架构设计，主要包括以下层次：

```
┌─────────────────────────────────────────────────┐
│                   UI 层                         │
│  (PySide6 GUI, 自定义控件, 样式管理)             │
├─────────────────────────────────────────────────┤
│                   业务逻辑层                     │
│  (设备管理, 报警管理, 数据处理)                  │
├─────────────────────────────────────────────────┤
│                   通信层                         │
│  (通信驱动, 协议实现)                            │
├─────────────────────────────────────────────────┤
│                   数据层                         │
│  (数据库模型, ORM, 数据存储与查询)                │
└─────────────────────────────────────────────────┘
```

## 6. 系统流程

### 6.1 设备添加流程

1. 用户在主窗口点击"添加设备"按钮
2. 弹出添加设备对话框
3. 用户填写设备信息，包括设备名称、类型、通信参数等
4. 用户选择是否启用自动重连
5. 点击"保存"按钮
6. 设备信息保存到数据库
7. 设备管理器创建设备实例
8. 设备开始定时采集数据

### 6.2 设备通信流程

1. 设备管理器初始化设备
2. 根据设备配置选择通信驱动
3. 建立通信连接
4. 定时发送采集命令
5. 接收设备响应数据
6. 解析数据并更新设备状态
7. 处理通信异常，根据配置决定是否重连

### 6.3 自动重连流程

1. 设备通信断开
2. 检查设备的自动重连配置
3. 如果启用自动重连，等待重连间隔时间
4. 尝试重新建立通信连接
5. 连接成功，恢复正常通信
6. 连接失败，继续尝试或放弃

## 8. 部署和运行

### 8.1 环境要求

- Python 3.8+
- PySide6
- SQLAlchemy
- 其他依赖见requirements.txt

### 8.2 运行方式

```bash
# 安装依赖
pip install -r requirements_v2.txt

```
