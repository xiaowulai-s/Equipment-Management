# 工业设备管理系统 v2.0 - 新架构说明

## 概述

工业设备管理系统 v2.0 基于四层解耦架构重构，采用插件化设计，支持多设备并发管理和实时数据监控。

## 四层解耦架构

### 1. 通信驱动层 (Communication Driver Layer)

**位置**: `core/communication/`

**职责**:
- 提供底层I/O抽象
- 处理字节流缓冲区逻辑
- 解决工业现场的粘包、断包问题

**组件**:
- `base_driver.py` - 通信驱动基类
- `tcp_driver.py` - TCP驱动（支持异步心跳）
- `serial_driver.py` - 串口驱动（基于pyserial，支持RS485）

### 2. 协议插件层 (Protocol Plugin Layer)

**位置**: `core/protocols/`

**职责**:
- 插件化设计，易于扩展
- 定义统一的协议抽象接口
- 处理协议帧打包/解包

**组件**:
- `base_protocol.py` - 协议基类
- `modbus_protocol.py` - Modbus协议实现（支持TCP/RTU，FC 03/04/06/10）

**Modbus特性**:
- 支持RTU和TCP模式
- 内置CRC16校验
- 自动重试机制（3次，间隔500ms）

### 3. 设备管理服务层 (Device Service Layer)

**位置**: `core/device/`

**职责**:
- 设备工厂：根据协议类型动态创建设备
- 设备管理：多设备生命周期管理
- 配置持久化：config.json自动保存/加载
- 模拟器：内置仿真数据生成

**组件**:
- `device_model.py` - 设备模型和状态枚举
- `device_factory.py` - 设备工厂
- `device_manager.py` - 设备管理器
- `simulator.py` - 设备模拟器

**特性**:
- 每个设备独立线程
- 毫秒级定时采集
- 指令队列管理
- 自动配置持久化

### 4. UI表现层 (UI Presentation Layer)

**位置**: `main_v2.py`

**职责**:
- 动态表单生成器
- 设备列表管理
- 实时数据卡片展示
- 设备状态监控

**特性**:
- 基于PySide6的原生Widgets
- 动态参数表单生成
- 设备树状列表
- 实时数据卡片网格

## 模拟器 (Simulator)

### 功能
- 生成符合物理规律的随机波动数据
- 支持温度、压力、流量等传感器模拟
- 内置报警逻辑（超温、超压）
- 日周期波动 + 随机噪声

### 模拟数据
- **温度**: 25°C ± 2°C，日周期波动
- **压力**: 1.0MPa ± 0.5MPa，快慢波动
- **流量**: 50.0m³/h ± 10m³/h，与压力相关
- **报警**: 温度>40°C或压力>1.2MPa触发

## 快速开始

### 1. 安装依赖

```bash
pip install -r requirements_v2.txt
```

### 2. 运行程序

```bash
python main_v2.py
```

### 3. 添加设备

1. 点击左侧"+"按钮
2. 输入设备名称
3. 选择协议类型（Modbus TCP/RTU）
4. 勾选"启用仿真模式"（推荐用于测试）
5. 点击"确定"

### 4. 连接设备

1. 在设备列表中选择设备
2. 点击"连接"按钮
3. 查看右侧实时数据卡片

## 配置文件

### config.json

系统自动生成和维护的设备配置文件。

```json
{
  "version": "1.0",
  "devices": [
    {
      "device_id": "abc12345",
      "name": "测试设备",
      "protocol_type": "modbus_tcp",
      "use_simulator": true,
      "host": "127.0.0.1",
      "port": 502,
      "unit_id": 1,
      "register_map": [...]
    }
  ]
}
```

## 设计原则

### 高内聚、低耦合
- 每层职责单一明确
- 层间通过信号与槽通信
- 易于扩展和维护

### 插件化
- 协议插件易于添加新协议
- 设备工厂支持动态创建
- 配置驱动，无需硬编码

### 工业级
- 内置重试和错误处理
- 配置持久化和自动恢复
- 完整的日志和状态监控
