# 工业设备管理系统 - 项目总览文档

## 一、项目简介

工业设备管理系统是一个基于Python + PySide6 + QML构建的现代化工业监控上位机软件，用于管理和监控多种工业设备。系统采用配置驱动、分层架构和插件化设计，支持多设备并发管理、实时数据采集和可视化展示。

### 核心特性
- ✅ 配置驱动架构 - 产品差异通过配置文件实现
- ✅ 分层架构设计 - 五层架构，职责清晰
- ✅ 多设备并发管理 - 支持同时管理多个设备
- ✅ Modbus TCP通信 - 完整的Modbus协议支持
- ✅ 实时数据可视化 - 仪表盘、趋势图、数据卡片
- ✅ 工业级UI设计 - 深色主题，符合Windows 11 Fluent Design
- ✅ 商业级日志系统 - 完整的日志记录和查询
- ✅ 数据持久化 - SQLite数据库存储历史数据

## 二、系统架构

### 2.1 五层架构模型

```
┌──────────────────────────────┐
│        用户界面层 (UI)         │
│  PySide6 + QML               │
└──────────────┬───────────────┘
               │
┌──────────────▼───────────────┐
│       业务逻辑控制层          │
│  设备管理 / 数据处理 / 命令调度  │
└──────────────┬───────────────┘
               │
┌──────────────▼───────────────┐
│       设备模型与配置层         │
│  产品Profile / 寄存器映射      │
└──────────────┬───────────────┘
               │
┌──────────────▼───────────────┐
│       协议与通信驱动层         │
│  Modbus TCP / 分包合并逻辑     │
└──────────────┬───────────────┘
               │
┌──────────────▼───────────────┐
│     数据持久化与日志层         │
│  SQLite / 通信日志 / 历史记录   │
└──────────────────────────────┘
```

### 2.2 各层职责说明

#### 用户界面层 (UI Layer)
- **技术选型**：PySide6 + QML
- **核心职责**：
  - 多窗口/多标签管理
  - 动态渲染界面元素
  - 实时数据展示（表格、曲线、仪表盘）
- **界面风格**：现代工业风（Fluent Design），深色主题

#### 业务逻辑控制层 (Controller Layer)
- **核心职责**：
  - 设备生命周期管理
  - 任务调度器
  - 命令处理
  - 数据处理
- **设计亮点**：状态管理、错误处理、事件驱动

#### 设备模型与配置层 (Device & Config Layer)
- **产品Profile**：JSON配置文件定义变量与寄存器映射
- **设备抽象层**：统一的设备接口

#### 协议与通信驱动层 (Protocol & Comm Layer)
- **技术选型**：pymodbus
- **核心设计**：Modbus TCP引擎、分包合并、连接池管理

#### 数据持久化与日志层 (Data Layer)
- **SQLite数据库**：存储历史数据
- **日志引擎**：分级日志、日志轮转

## 八、快速开始

### 8.1 安装依赖

```bash
pip install -r requirements.txt
```

### 8.2 运行程序

```bash
python main.py
```

```bash

```
