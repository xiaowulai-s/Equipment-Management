## 二、代码架构（分层设计）

### 2.1 四层解耦架构 (已完整实现)

```
┌─────────────────────────────────────────────────────────────┐
│                    用户界面层 (UI Layer)                      │
│  PySide6 Widgets                                            │
│  • MainWindow (主窗口)                                      │
│  • AddDeviceDialog (添加设备)                              │
│  • DeviceTypeDialog (设备类型管理)                         │
│  • AppStyles (统一样式)                                    │
├─────────────────────────────────────────────────────────────┤
│                    业务逻辑层 (Business Layer)               │
│  main.py                                                    │
│  • 窗口布局管理                                             │
│  • 信号槽连接                                              │
│  • 用户交互处理                                            │
│  • 报警规则设置                                             │
├─────────────────────────────────────────────────────────────┤
│                    设备管理层 (Device Management)            │
│  DeviceManager (QObject + Signals)                         │
│  • 设备生命周期管理                                         │
│  • 设备状态监控                                             │
│  • 设备轮询控制                                            │
├─────────────────────────────────────────────────────────────┤
│                    协议与通信层 (Protocol & Comm)           │
│  ModbusProtocol / TcpDriver / SerialDriver                │
│  • Modbus TCP/RTU通信                                      │
│  • 驱动可插拔设计                                           │
├─────────────────────────────────────────────────────────────┤
│                    数据持久化层 (Data Layer)                │
│  SQLite / JSON配置 / 日志系统                              │
│  • AlarmManager (报警系统)                                 │
│  • DataExporter (数据导出)                                 │
└─────────────────────────────────────────────────────────────┘
```

```python
# 报警级别

AlarmType.COMMUNICATION_ERROR  # 通信错误
```
