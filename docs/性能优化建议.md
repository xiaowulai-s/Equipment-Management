## 优化建议

### 一、UI层优化

#### 1.1 DataCard组件优化

**当前状态**:
- 创建耗时: ~5-15ms
- 更新耗时: ~0.5-2ms

**优化建议**:

1. **延迟初始化**
   - 仅在实际需要显示时创建内部控件
   - 使用QLayout的sizePolicy优化布局计算

2. **缓存QPixmap**
   - 状态图标（正常/警告/错误）应缓存为QPixmap
   - 避免每帧重新绘制图标

3. **减少样式表重应用**
   - 只在状态变化时更新QSS
   - 使用`setProperty()` + QSS选择器替代直接设置样式

**代码示例**:
```python
class DataCard(QWidget):
    def __init__(self, *args, **kwargs):
        super().__init__(*args, **kwargs)
        # 缓存图标
        self._icon_cache = {
            'normal': self._load_icon('normal.png'),
            'warning': self._load_icon('warning.png'),
            'error': self._load_icon('error.png'),
        }

    def update_status(self, status: str):
        # 仅在状态变化时更新
        if self._current_status != status:
            self.setProperty('status', status)
            self.style().unpolish(self)
            self.style().polish(self)
            self._current_status = status
```

#### 1.2 Gauge组件优化

**当前状态**:
- Canvas绘制: ~10-20ms/帧
- 更新频率: >30FPS时可能卡顿

**优化建议**:

1. **降低绘制分辨率**
   - 根据`devicePixelRatio()`调整绘制质量
   - 高DPI设备可降低分辨率以提升性能

2. **使用离屏缓存**
   - 对于静态背景（刻度盘），绘制到QPixmap并缓存
   - 仅更新指针部分

3. **限制更新频率**
   - 使用定时器限制最大更新频率（如30FPS）
   - 值未变化时跳过绘制

**代码示例**:
```python
class Gauge(QWidget):
    def __init__(self, *args, **kwargs):
        super().__init__(*args, **kwargs)
        self._last_value = None
        self._last_update_time = 0
        self._min_update_interval = 33  # 30 FPS
        self._background_cache = None

    def set_value(self, value: float):
        # 值未变化且未超过最小间隔时跳过
        current_time = time.time() * 1000
        if (self._last_value == value and
            current_time - self._last_update_time < self._min_update_interval):
            return

        self._value = value
        self._last_value = value
        self._last_update_time = current_time
        self.update()  # 触发重绘

    def paintEvent(self, event):
        painter = QPainter(self)
        painter.setRenderHint(QPainter.Antialiasing)

        # 使用缓存的背景
        if self._background_cache is None:
            self._background_cache = self._draw_background()
        painter.drawPixmap(0, 0, self._background_cache)

        # 仅绘制指针
        self._draw_needle(painter)
```

#### 1.3 TrendChart组件优化

**当前状态**:
- 数据点 > 500时性能下降
- 每次更新都触发完整重绘

**优化建议**:

1. **数据采样/降采样**
   - 当数据点超过阈值时，进行降采样
   - 保留关键点（极值点、转折点）

2. **使用QGraphicsView替代Canvas**
   - QGraphicsView提供更好的缩放/平移性能
   - 支持GPU加速

3. **分层数据加载**
   - 缩放级别低时显示降采样数据
   - 缩放级别高时加载完整数据

4. **虚拟滚动**
   - 仅渲染视口内的数据点
   - 使用QGraphicsView的clip功能

**代码示例**:

#### 1.4 RealTimeChart组件优化

**当前状态**:
- pyqtgraph性能良好（设计合理）
- 大数据点（>10K）时内存增长明显

**优化建议**:

1. **固定数据窗口**
   - 使用`setDownsampling()`启用降采样
   - 使用`setClipToView(True)`仅绘制可见区域

2. **使用numpy数组**
   - 替换Python列表为numpy数组
   - 减少数据拷贝次数

3. **批量更新**
   - 积累多个数据点后批量添加
   - 使用`setData()`替代`addPoint()`

**代码示例**:
```python
class RealTimeChart(pg.PlotWidget):
    def __init__(self, *args, **kwargs):
        super().__init__(*args, **kwargs)
        self.setDownsampling(auto=True, mode='peak')
        self.setClipToView(True)
        self._data_buffers = {}  # {series_name: {'x': [], 'y': []}}
        self._batch_size = 100

    def add_point(self, name: str, x: float, y: float):
        if name not in self._data_buffers:
            self._data_buffers[name] = {'x': [], 'y': []}

        buffer = self._data_buffers[name]
        buffer['x'].append(x)
        buffer['y'].append(y)

        # 批量更新
        if len(buffer['x']) >= self._batch_size:
            self._flush_buffer(name)

    def _flush_buffer(self, name: str):
        buffer = self._data_buffers[name]
        if not buffer['x']:
            return

        curve = self.curves[name]
        x_data = np.array(buffer['x'])
        y_data = np.array(buffer['y'])

        # 获取现有数据并追加
        existing_x, existing_y = curve.getData()
        if existing_x is not None:
            x_data = np.concatenate([existing_x, x_data])
            y_data = np.concatenate([existing_y, y_data])

        # 更新曲线
        curve.setData(x_data, y_data)

        # 清空缓冲区
        buffer['x'].clear()
        buffer['y'].clear()
```

### 二、数据层优化

#### 2.1 设备管理器优化

**当前状态**:
- 100设备 × 200寄存器 ≈ 20000寄存器
- 内存占用: ~50-100MB
- 随机访问: ~0.5-2ms

**优化建议**:

1. **使用__slots__减少内存**
   - 在Register类中启用`__slots__`
   - 减少对象字典开销

2. **延迟加载寄存器**
   - 仅在需要时创建Register对象
   - 使用寄存器模板按需实例化

3. **寄存器分页**
   - 将寄存器分组（如按地址范围）
   - 仅加载当前页面的寄存器

**代码示例**:
```python
from dataclasses import dataclass

@dataclass(slots=True)  # Python 3.10+
class Register:
    __slots__ = [
        'name', 'address', 'register_type', 'data_type',
        'value', 'alarm_config', '_cached_alarm',
    ]

    # ... rest of the implementation
```

#### 2.2 数据采集引擎优化

**当前状态**:
- PollWorker使用QThread
- 数据更新延迟: ~100-500ms

**优化建议**:

1. **请求合并**
   - 将连续的寄存器读取请求合并
   - 减少Modbus请求次数

2. **异步I/O**
   - 使用asyncio替代QThread（未来方向）
   - 非阻塞I/O减少线程数量

3. **批量写入**
   - 积累多个写操作后批量提交
   - 减少数据库事务开销

### 三、内存管理优化

#### 3.1 对象生命周期管理

**常见问题**:
- QWidget对象未正确释放
- 信号槽连接未断开
- QTimer未停止

**优化建议**:

1. **显式释放资源**
   - 在析构函数中清理资源
   - 断开所有信号槽连接

2. **使用QObject.deleteLater()**
   - 延迟删除对象，确保事件循环处理完毕
   - 避免删除使用中的对象

**代码示例**:
```python
class CustomWidget(QWidget):
    def __init__(self, parent=None):
        super().__init__(parent)
        self._timer = QTimer(self)

    def cleanup(self):
        """清理资源"""
        # 停止定时器
        if self._timer.isActive():
            self._timer.stop()

        # 断开所有信号连接
        try:
            self.disconnect()
        except:
            pass

    def closeEvent(self, event):
        """窗口关闭时清理"""
        self.cleanup()
        super().closeEvent(event)
```

---

#### 3.2 垃圾回收策略

**优化建议**:

1. **手动触发gc.collect()**
   - 在数据更新间隙调用
   - 定期清理循环引用

2. **避免循环引用**
   - 使用weakref引用父对象
   - 避免在lambda中捕获self

**代码示例**:
```python
import weakref

class DeviceManager(QObject):
    def __init__(self, parent=None):
        super().__init__(parent)
        # 使用弱引用避免循环引用
        self._device_tree = weakref.WeakValueDictionary()

    def connect_device_signals(self, device):
        # 断开旧连接
        try:
            device.data_updated.disconnect()
        except:
            pass

        # 使用lambda避免循环引用
        device.data_updated.connect(
            lambda data, d=device: self._on_data_updated(d, data)
        )
```

### 四、数据库优化

#### 4.1 SQLite优化

**优化建议**:

1. **批量插入**
   - 使用executemany()批量插入
   - 减少事务提交次数

2. **索引优化**
   - 为常用查询字段创建索引
   - 复合索引覆盖多列

3. **WAL模式**
   - 已启用（良好实践）
   - 定期checkpoint

**代码示例**:
```python
def batch_insert_historical_data(self, records: List[HistoricalRecord]):
    """批量插入历史数据"""
    with self.session() as session:
        # 批量插入
        session.execute(
            insert(HistoricalDataModel).values(
                [r.to_dict() for r in records]
            )
        )
        session.commit()

        # 定期checkpoint
        if random.random() < 0.1:  # 10%几率
            self._connection.execute("PRAGMA wal_checkpoint(TRUNCATE)")
```

### 五、并发优化

#### 5.1 线程管理

**当前状态**:
- 每个设备一个PollWorker线程
- 100设备 ≈ 100线程

**优化建议**:

1. **线程池**
   - 使用QThreadPool管理线程
   - 限制最大线程数（如CPU核心数×2）

2. **协程（asyncio）**
   - 未来迁移到asyncio
   - 单线程处理多设备

**代码示例**:
```python
from PySide6.QtCore import QThreadPool, QRunnable

class DataCollector(QObject):
    def __init__(self, parent=None):
        super().__init__(parent)
        self._thread_pool = QThreadPool()
        self._thread_pool.setMaxThreadCount(8)  # 限制线程数

    def poll_device(self, device: Device):
        task = PollTask(device)
        self._thread_pool.start(task)

class PollTask(QRunnable):
    def __init__(self, device):
        super().__init__()
        self.device = device
```
