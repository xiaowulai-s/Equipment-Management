## 安装方式

### 方式一：使用PyInstaller打包的独立可执行文件

这是最简单的安装方式，无需配置Python环境。

#### Windows

1. **下载安装包**
   - 从发布页面下载 `EquipmentManagement-vX.X.X-win.zip`
   - 解压到任意目录（如 `C:\Program Files\EquipmentManagement`）

2. **运行程序**
   - 双击 `EquipmentManagement.exe` 启动程序

3. **（可选）创建快捷方式**
   - 右键 `EquipmentManagement.exe`
   - 选择 "发送到" -> "桌面快捷方式"

#### macOS

1. **下载安装包**
   - 从发布页面下载 `EquipmentManagement-vX.X.X-mac.zip`
   - 解压到应用程序目录

2. **运行程序**
   - 双击 `EquipmentManagement` 启动程序

3. **（可选）解决安全警告**
   - 如果遇到"无法打开"的警告
   - 右键点击应用，选择"打开"
   - 在安全对话框中点击"打开"

#### Linux

1. **下载安装包**
   - 从发布页面下载 `EquipmentManagement-vX.X.X-linux.tar.gz`
   - 解压到任意目录：
     ```bash
     tar -xzf EquipmentManagement-vX.X.X-linux.tar.gz
     cd EquipmentManagement
     ```

2. **运行程序**
   ```bash
   ./EquipmentManagement
   ```

3. **（可选）创建桌面快捷方式**
   - 创建 `.desktop` 文件：
     ```ini
     [Desktop Entry]
     Name=Equipment Management
     Exec=/path/to/EquipmentManagement
     Icon=/path/to/icon.png
     Type=Application
     Categories=Development;
     ```
   - 复制到 `~/.local/share/applications/`

### 方式二：从源代码安装

这种方式适用于开发环境或需要自定义配置。

#### 1. 安装Python

确保安装了Python 3.10或更高版本：

```bash
python --version
```

如果没有安装，请从 [Python官网](https://www.python.org/downloads/) 下载安装。

#### 2. 克隆或下载源代码

```bash
git clone <repository-url>
cd equipment-management
```

#### 3. 创建虚拟环境（推荐）

```bash
# Windows
python -m venv venv
venv\Scripts\activate

# macOS/Linux
python3 -m venv venv
source venv/bin/activate
```

#### 4. 安装依赖

```bash
pip install -r requirements.txt
```

依赖列表包括：
- PySide6 >= 6.4.0
- SQLAlchemy >= 2.0
- pyserial >= 3.5
- pyqtgraph >= 0.13
- numpy >= 1.24

#### 5. 启动程序

```bash
python main.py
```

## 配置

### 配置文件位置

程序首次运行时会自动创建配置文件：

| 操作系统 | 配置文件路径 |
|----------|-------------|
| **Windows** | `%APPDATA%\EquipmentManagement\config.json` |
| **macOS** | `~/Library/Application Support/EquipmentManagement/config.json` |
| **Linux** | `~/.config/EquipmentManagement/config.json` |

### 配置项说明

```json
{
  "theme": "dark",
  "data": {
    "poll_interval": 1000,
    "max_retry": 3,
    "retry_delay": 1000
  },
  "alarm": {
    "enable_popup": true,
    "enable_sound": false,
    "cooldown_seconds": 60,
    "min_level": "warning"
  },
  "logging": {
    "level": "INFO",
    "enable_file_log": true,
    "log_path": "logs/"
  },
  "ui": {
    "language": "zh-CN",
    "font_family": "Segoe UI",
    "font_size": 12
  }
}
```

#### 主题设置
- `theme`: `dark`（深色）, `light`（浅色）, `system`（跟随系统）

#### 数据采集设置
- `poll_interval`: 轮询间隔（毫秒）
- `max_retry`: 最大重试次数
- `retry_delay`: 重试延迟（毫秒）

#### 报警设置
- `enable_popup`: 是否启用弹窗通知
- `enable_sound`: 是否启用声音通知
- `cooldown_seconds`: 报警冷却时间（秒）
- `min_level`: 最低通知级别（`info`, `warning`, `error`, `critical`）

#### 日志设置
- `level`: 日志级别（`DEBUG`, `INFO`, `WARNING`, `ERROR`, `CRITICAL`）
- `enable_file_log`: 是否启用文件日志
- `log_path`: 日志文件路径

#### 界面设置
- `language`: 界面语言（`zh-CN`, `en-US`）
- `font_family`: 字体家族
- `font_size`: 字体大小

### 修改配置

有两种方式修改配置：

1. **通过界面设置**
   - 点击菜单栏 "工具" -> "设置"
   - 在对话框中修改配置并应用

2. **直接编辑配置文件**
   - 使用文本编辑器打开配置文件
   - 修改后保存，程序会自动重新加载

## 故障排除

### 问题1：程序无法启动

**Windows:**
- 检查是否安装了Visual C++ Redistributable
- 尝试以管理员身份运行

**macOS:**
- 如果提示"无法打开"，右键点击 -> 打开
- 检查系统安全设置

**Linux:**
- 检查可执行文件权限：`chmod +x EquipmentManagement`
- 检查是否安装了必要的系统库

### 问题2：设备连接失败

1. **检查网络连接**
   ```bash
   ping <设备IP地址>
   ```

2. **检查防火墙设置**
   - 确保Modbus TCP端口（默认502）未被阻止

3. **检查设备配置**
   - 确认设备IP地址、端口、从站地址配置正确
   - 确认设备Modbus参数（功能码、寄存器地址）正确

4. **查看日志**
   - 日志文件位于配置文件中的`log_path`路径
   - 查找错误信息和堆栈跟踪

### 问题3：串口设备无法识别

**Windows:**
1. 打开设备管理器，查看"端口(COM和LPT)"
2. 记录COM口号（如COM3）
3. 在程序配置中使用该COM口号

**Linux:**
1. 列出可用串口：
   ```bash
   ls /dev/ttyUSB* /dev/ttyACM*
   ```
2. 确保用户有访问权限：
   ```bash
   sudo usermod -a -G dialout $USER
   ```

**macOS:**
1. 列出可用串口：
   ```bash
   ls /dev/cu.*
   ```
2. 安装驱动（如CH340驱动）

### 问题4：界面卡顿

1. **减少监控的设备数量**
   - 暂停不必要的设备监控

2. **调整轮询间隔**
   - 在设置中增加轮询间隔（如从1000ms改为2000ms）

3. **检查系统资源**
   - 打开任务管理器/活动监视器
   - 查看CPU和内存使用情况

### 问题5：数据库错误

1. **删除旧数据库**
   - 数据库文件位于配置目录
   - 删除`equipment_management.db`文件
   - 程序会自动重建数据库

2. **检查磁盘空间**
   - 确保有足够的磁盘空间

## 日志文件

日志文件记录了程序的运行状态和错误信息，对故障排除非常重要。

### 日志位置

| 操作系统 | 日志路径 |
|----------|---------|
| **Windows** | `%APPDATA%\EquipmentManagement\logs\` |
| **macOS** | `~/Library/Logs/EquipmentManagement\` |
| **Linux** | `~/.local/share/EquipmentManagement/logs\` |

### 日志文件说明

- `equipment_management.log`: 主日志文件
- `equipment_management_debug.log`: 调试日志（仅在开发模式下）
- `equipment_management_error.log`: 错误日志

### 查看日志

```bash
# Windows
type %APPDATA%\EquipmentManagement\logs\equipment_management.log

# macOS/Linux
tail -f ~/.local/share/EquipmentManagement/logs/equipment_management.log
```

## 卸载

### Windows

1. **删除程序文件**
   - 删除安装目录
   - 删除桌面快捷方式

2. **删除配置和数据**
   - 删除 `%APPDATA%\EquipmentManagement` 目录

3. **删除注册表项（可选）**
   ```cmd
   reg delete "HKEY_CURRENT_USER\Software\EquipmentManagement" /f
   ```

### macOS

```bash
# 删除应用程序
rm -rf /Applications/EquipmentManagement.app

# 删除配置和数据
rm -rf ~/Library/Application\ Support/EquipmentManagement

# 删除日志
rm -rf ~/Library/Logs/EquipmentManagement
```

### Linux

```bash
# 删除程序文件
rm -rf /path/to/EquipmentManagement

# 删除配置和数据
rm -rf ~/.config/EquipmentManagement

# 删除日志
rm -rf ~/.local/share/EquipmentManagement/logs

# 删除桌面快捷方式（如果有）
rm ~/.local/share/applications/equipment-management.desktop
```
