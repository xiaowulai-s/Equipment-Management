## 简介

工业设备管理系统是一款基于 PySide6 和 Modbus 协议的上位机软件，用于监控和管理工业现场设备。

### 主要特性

- ✅ 支持 Modbus TCP、Modbus RTU、Modbus ASCII 三种协议
- ✅ 多设备并发管理
- ✅ 实时数据监控和历史记录
- ✅ 可视化报警规则配置
- ✅ 浅色/深色主题切换
- ✅ 数据导出功能（CSV、Excel）

### 系统架构

```
┌─────────────────┐
│    UI 层        │  PySide6 Widgets
├─────────────────┤
│  设备管理层     │  DeviceManager
├─────────────────┤
│   通信层        │  TCP/Serial Driver
├─────────────────┤
│   协议层        │  Modbus Protocol
└─────────────────┘
```

## 安装指南

### 步骤 1：安装 Python

1. 访问 [Python 官网](https://www.python.org/downloads/)
2. 下载并安装 Python 3.9+
3. 安装时勾选 "Add Python to PATH"

### 步骤 2：克隆项目

```bash
git clone https://github.com/xiaowulai-s/Equipment-Management.git
cd equipment management
```

### 步骤 3：创建虚拟环境

```bash
# Windows
python -m venv .venv
.venv\Scripts\activate

# Linux/Mac
python3 -m venv venv
source venv/bin/activate
```

### 步骤 4：安装依赖

```bash
pip install -r requirements.txt
```

依赖包包括：
- PySide6 - Qt for Python
- SQLAlchemy - 数据库 ORM
- pyserial - 串口通信
- pandas - 数据处理
- openpyxl - Excel 支持

### 步骤 5：运行程序

```bash
python main.py
```

## 快速入门

### 5.1 启动程序

双击 `main.py` 或在命令行运行：

```bash
python main.py
```

程序启动后将显示主窗口。

### 5.2 添加第一个设备

1. 点击工具栏的 **"添加设备"** 按钮
2. 填写设备信息：
   - **设备名称**: 例如 "PLC-001"
   - **设备类型**: 选择 "PLC"
   - **协议类型**: 选择 "Modbus TCP"
   - **IP 地址**: 输入设备 IP，如 "192.168.1.100"
   - **端口**: 输入 502
   - **从站 ID**: 输入 1
3. 点击 **"测试连接"** 验证通信
4. 点击 **"确定"** 保存

### 5.3 连接设备

在设备列表中，点击设备行的 **"连接"** 按钮，设备状态将变为 "已连接"。

### 5.4 配置寄存器

1. 点击设备行的 **"编辑"** 按钮
2. 切换到 **"寄存器配置"** 标签
3. 点击 **"添加寄存器"**
4. 填写寄存器信息：
   - **地址**: 40001
   - **名称**: "温度"
   - **数据类型**: "Int16"
   - **读写类型**: "只读"
5. 点击 **"确定"** 保存

### 5.5 查看实时数据

连接设备后，主窗口将实时显示设备数据，包括：
- 设备状态（在线/离线）
- 寄存器数值
- 通信质量

## 功能使用

### 6.1 设备管理

#### 添加设备

支持添加多种类型的设备：
- **PLC**: 可编程逻辑控制器
- **Sensor**: 传感器（温湿度、压力等）
- **Meter**: 仪表（电表、水表等）
- **VFD**: 变频器
- **Generic**: 通用设备

#### 编辑设备

1. 在设备列表中点击 **"编辑"** 按钮
2. 修改设备配置
3. 点击 **"保存"** 应用更改

#### 删除设备

1. 在设备列表中点击 **"删除"** 按钮
2. 确认删除操作

**注意**: 删除设备将同时删除所有关联的寄存器配置和历史数据。

### 6.2 通信配置

#### Modbus TCP 配置

```
协议类型：Modbus TCP
IP 地址：192.168.1.100
端口：502
从站 ID: 1
超时时间：1000ms
```

#### Modbus RTU 配置

```
协议类型：Modbus RTU
串口号：COM1
波特率：9600
数据位：8
停止位：1
校验位：None
从站 ID: 1
```

#### 串口测试

在添加 Modbus RTU 设备时，可以点击 **"测试"** 按钮验证串口是否可用。

### 6.3 寄存器管理

#### 添加寄存器

支持多种数据类型：
- **Int16**: 16 位整数
- **Int32**: 32 位整数
- **Float**: 浮点数
- **UInt16**: 无符号 16 位整数
- **Bit**: 位

#### 批量读取

系统支持批量读取连续寄存器，提高通信效率。

### 6.4 报警系统

#### 配置报警规则

1. 点击 **"报警配置"** 按钮
2. 选择设备和参数
3. 设置报警条件：
   - **上限报警**: 值 > 阈值
   - **下限报警**: 值 < 阈值
   - **偏差报警**: 值偏离设定值
4. 设置报警级别：
   - **提示**: 一般信息
   - **警告**: 需要注意
   - **严重**: 立即处理
   - **紧急**: 危险状态

#### 查看报警

报警信息将显示在：
- 状态栏报警指示灯
- 报警列表窗口
- 历史记录

### 6.5 主题切换

点击右上角的状态栏按钮，可以在浅色和深色主题之间切换。

主题偏好会自动保存，下次启动时自动加载。

### 6.6 数据导出

支持导出以下数据：
- 设备配置
- 历史数据
- 报警记录

导出格式：
- CSV
- Excel (XLSX)

### 7.1 配置文件

#### 设备配置文件

位置：`config/devices.json`


位置：`config/theme.json`

```json
{
  "theme": "light"
}
```

日志文件位置：`logs/app.log`

日志级别：
- **DEBUG**: 调试信息
- **INFO**: 一般信息
- **WARNING**: 警告信息
- **ERROR**: 错误信息

默认使用 SQLite 数据库：`data/equipment.db`

- `alarms`: 报警记录

## 常见问题

### Q1: 无法连接到设备

**可能原因**:
1. IP 地址或端口错误
2. 设备未开机
3. 网络不通
4. 防火墙阻止

**解决方法**:
1. 检查设备 IP 和端口配置
2. 使用 ping 命令测试网络连通性
3. 检查防火墙设置
4. 使用 "测试连接" 功能诊断

### Q2: 串口无法打开

**可能原因**:
1. 串口号错误
2. 串口被其他程序占用
3. 串口硬件故障

**解决方法**:
1. 检查串口号（设备管理器中查看）
2. 关闭占用串口的其他程序
3. 使用 "串口测试" 功能验证

### Q3: 读取的数据不正确

**可能原因**:
1. 寄存器地址错误
2. 数据类型不匹配
3. 字节顺序问题

**解决方法**:
1. 核对寄存器地址
2. 检查数据类型配置
3. 尝试不同的字节顺序（ABCD/BADC 等）

### Q4: 程序运行缓慢

**可能原因**:
1. 设备数量过多
2. 轮询间隔太短
3. 历史数据过多

**解决方法**:
1. 优化设备配置
2. 增加轮询间隔
3. 定期清理历史数据

### Q5: 如何备份数据

**方法**:
1. 备份 `data/equipment.db` 数据库文件
2. 备份 `config/` 配置目录
3. 导出重要数据为 CSV/Excel

## 附录

### A. Modbus 寄存器地址对照表

| 地址范围 | 类型 | 说明 |
|---------|------|------|
| 00001-09999 | Coil | 数字量输出 |
| 10001-19999 | Contact | 数字量输入 |
| 30001-39999 | Input Register | 模拟量输入 |
| 40001-49999 | Holding Register | 模拟量输出 |

### B. 数据类型说明

| 类型 | 大小 | 范围 | 说明 |
|------|------|------|------|
| Int16 | 2 字节 | -32768 ~ 32767 | 有符号 16 位整数 |
| Int32 | 4 字节 | -2147483648 ~ 2147483647 | 有符号 32 位整数 |
| UInt16 | 2 字节 | 0 ~ 65535 | 无符号 16 位整数 |
| Float | 4 字节 | ±3.4E38 | IEEE 754 浮点数 |
