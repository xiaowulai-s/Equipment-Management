## 简介

工业设备管理系统是一款基于 PySide6 和 Modbus 协议的上位机软件，用于监控和管理工业现场设备。采用四层解耦架构，具备高性能、高可靠性和良好的可扩展性。

### 主要特性

- ✅ 支持 Modbus TCP、Modbus RTU、Modbus ASCII 三种协议
- ✅ 多设备并发管理（支持100+设备）
- ✅ 实时数据监控与可视化
- ✅ pyqtgraph 高性能实时曲线图
- ✅ 四级报警系统（高高/高/低/低低）+ 死区控制
- ✅ Fluent Design 风格深色/浅色主题
- ✅ 数据持久化（SQLite + Repository 模式）
- ✅ 报警冷却机制，避免报警风暴
- ✅ 数据导出（CSV、Excel）

### 系统架构

```
┌─────────────────────────────────────────┐
│            UI 层 (PySide6 Widgets)       │
│  MainWindow + DataCard + Gauge + Chart   │
├─────────────────────────────────────────┤
│          设备管理层 (DeviceManager)       │
│  Device + Register + DataCollector       │
├─────────────────────────────────────────┤
│        通信驱动层 (TCP/Serial)           │
│  TcpDriver + SerialDriver + AutoReconn   │
├─────────────────────────────────────────┤
│         协议层 (Modbus Protocol)         │
│  TCP + RTU + ASCII + CRC/LRC校验         │
└─────────────────────────────────────────┘
          ↕
┌─────────────────────────────────────────┐
│        数据持久化层 (SQLite)              │
│  DatabaseManager + Repository模式        │
└─────────────────────────────────────────┘
```

## 安装指南

### 方式一：从可执行文件安装（推荐）

1. 从发布页面下载对应平台的安装包
2. 解压到任意目录
3. 双击运行 `EquipmentManagement.exe`（Windows）

详见 [安装部署指南](安装部署指南.md)。

### 方式二：从源代码安装

#### 步骤 1：安装 Python

1. 访问 [Python 官网](https://www.python.org/downloads/)
2. 下载并安装 Python 3.10+
3. 安装时勾选 "Add Python to PATH"

#### 步骤 2：克隆项目

```bash
git clone <repository-url>
cd equipment-management
```

#### 步骤 3：创建虚拟环境

```bash
# Windows
python -m venv .venv
.venv\Scripts\activate

# macOS/Linux
python3 -m venv venv
source venv/bin/activate
```

#### 步骤 4：安装依赖

```bash
pip install -r requirements.txt
```

核心依赖：
| 包 | 用途 |
|----|------|
| PySide6 | Qt for Python UI框架 |
| SQLAlchemy | 数据库ORM |
| pyserial | 串口通信 |
| pyqtgraph | 高性能数据可视化 |
| numpy | 数值计算 |

#### 步骤 5：运行程序

```bash
python main.py
```

## 快速入门

### 5.1 启动程序

```bash
python main.py
```

程序启动后显示主窗口，包含以下区域：

```
┌──────────────────────────────────────────────────┐
│  菜单栏  [设备] [工具] [视图] [帮助]              │
├──────────┬───────────────────────────────────────┤
│          │                                       │
│  设备树  │         监控页面                       │
│          │   ┌────────┐ ┌────────┐ ┌────────┐   │
│  ├ PLC   │   │DataCard│ │DataCard│ │DataCard│   │
│  ├ 传感器│   └────────┘ └────────┘ └────────┘   │
│  ├ 变频器│   ┌──────────────────────────────┐   │
│  └ 其他  │   │     RealTimeChart            │   │
│          │   │     实时曲线图               │   │
│          │   └──────────────────────────────┘   │
├──────────┴───────────────────────────────────────┤
│  状态栏: 在线设备: 5/10  |  报警: 2  |  收集率 98%│
└──────────────────────────────────────────────────┘
```

### 5.2 添加第一个设备

1. 点击菜单栏 **"设备" → "添加设备"** 或点击工具栏的 "+" 按钮
2. 填写设备信息：
   - **设备名称**: 例如 "PLC-001"
   - **协议类型**: 选择 "Modbus TCP"
   - **IP 地址**: 输入设备 IP，如 "192.168.1.100"
   - **端口**: 输入 502
   - **从站 ID**: 输入 1
   - **超时**: 5000ms
   - **最大重试**: 3
3. 点击 **"确定"** 保存

### 5.3 配置寄存器

1. 在设备树中选择设备，点击 **"编辑"** 按钮
2. 切换到 **"寄存器配置"** 选项卡
3. 点击 **"添加寄存器"**
4. 填写寄存器信息：
   - **名称**: "温度"
   - **地址**: 0（对应40001）
   - **寄存器类型**: "Holding Register"（保持寄存器）
   - **数据类型**: "Float"
   - **报警配置**（可选）:
     - 高高报警: 100.0
     - 高报警: 80.0
     - 低报警: 20.0
     - 低低报警: 10.0
     - 死区: 2.0
5. 点击 **"确定"** 保存

### 5.4 连接设备

1. 在设备树中选中设备
2. 点击 **"连接"** 按钮
3. 设备状态指示灯变为绿色（在线）
4. 监控页面自动显示实时数据卡片和曲线图

### 5.5 查看实时数据

连接设备后，监控页面将展示：

| 组件 | 说明 |
|------|------|
| **DataCard** | 数据卡片，显示寄存器值、单位、状态 |
| **Gauge** | 仪表盘，可视化显示关键参数（0-100%刻度） |
| **RealTimeChart** | 实时曲线图，显示最近60秒数据趋势 |

## 功能使用

### 6.1 设备管理

#### 添加设备

支持通过菜单栏或工具栏添加设备。填写表单即可：

- **Modbus TCP**: 需要填写 IP 地址、端口、从站 ID
- **Modbus RTU**: 需要填写串口号、波特率、数据位、停止位、校验位
- **Modbus ASCII**: 同 RTU 参数 + ASCII 编码模式

#### 编辑设备

1. 在设备树中选中设备
2. 点击 **"编辑"** 按钮
3. 修改配置后保存

#### 删除设备

1. 在设备树中选中设备
2. 点击 **"删除"** 按钮
3. 确认删除（将同时删除关联的寄存器配置和历史数据）

#### 批量操作

通过 **"工具" → "批量操作"** 可以：
- 批量连接/断开设备
- 批量导出设备配置
- 批量删除设备

### 6.2 寄存器管理

#### 寄存器类型

| 类型 | 说明 | 功能码 | 地址范围 |
|------|------|--------|----------|
| **Coil** | 线圈（读写） | 01/05/15 | 0-65535 |
| **Discrete Input** | 离散输入（只读） | 02 | 0-65535 |
| **Input Register** | 输入寄存器（只读） | 04 | 0-65535 |
| **Holding Register** | 保持寄存器（读写） | 03/06/16 | 0-65535 |

#### 数据类型

| 类型 | 大小 | 范围 | 说明 |
|------|------|------|------|
| **BOOL** | 1 bit | 0/1 | 位操作 |
| **UINT16** | 2 字节 | 0 ~ 65535 | 无符号16位整数 |
| **INT16** | 2 字节 | -32768 ~ 32767 | 有符号16位整数 |
| **UINT32** | 4 字节 | 0 ~ 4294967295 | 无符号32位整数 |
| **INT32** | 4 字节 | -2^31 ~ 2^31-1 | 有符号32位整数 |
| **FLOAT** | 4 字节 | ±3.4E38 | IEEE 754浮点数 |

### 6.3 报警系统

#### 四级报警机制

系统采用四级报警机制，支持死区控制以防止频繁报警：

```
                     高高报警 (HH)
                         │
                     高报警 (H)
                         │
                   ┌─── 死区 ───┐
                   │            │
         ┌─────────┘            └─────────┐
      低报警 (L)              高报警 (H)
         │                        │
     低低报警 (LL)              高高报警 (HH)
```

#### 报警级别

| 级别 | 颜色 | 说明 |
|------|------|------|
| **INFO** | 蓝色 | 一般信息 |
| **WARNING** | 黄色 | 需要注意 |
| **ERROR** | 橙色 | 立即处理 |
| **CRITICAL** | 红色 | 危险状态 |

#### 报警冷却

同一设备同一寄存器在冷却时间内不会重复报警通知，默认冷却时间60秒。可在系统设置中调整。

#### 查看报警

报警信息显示在：
- 状态栏报警计数器
- 报警列表（菜单栏 → "工具" → "报警历史"）
- DataCard状态指示灯（黄色/红色闪烁）

#### 确认报警

在报警历史中，可以对报警记录进行确认操作，标记为已处理。

### 6.4 实时监控

#### 监控页面

监控页面包含两个选项卡：

**实时数据选项卡**：
- DataCard 网格布局：每个寄存器显示为一个数据卡片
- Gauge 仪表盘：关键参数（配置了报警的寄存器）以仪表盘显示
- 自动每秒更新数据

**历史趋势选项卡**：
- RealTimeChart 实时曲线图
- 支持多系列数据同时显示
- 自动滚动显示最近数据
- 支持缩放和平移

#### 数据卡片说明

```
┌─────────────────────┐
│ 🟢 温度     Normal  │  ← 状态指示灯 + 寄存器名称
│                     │
│     25.5 °C         │  ← 当前值 + 单位
│                     │
│  ↑ +0.3  ▼ 20~80   │  ← 趋势指示 + 报警范围
└─────────────────────┘
```

## 高级配置

### 8.1 配置文件

配置文件路径（首次运行自动创建）：

| 操作系统 | 路径 |
|----------|------|
| **Windows** | `%APPDATA%\EquipmentManagement\config.json` |

### 8.2 数据库

默认使用 SQLite 数据库，位于配置目录下 `equipment_management.db`。

数据库表结构：

| 表名 | 说明 |
|------|------|
| `devices` | 设备基本信息 |
| `register_maps` | 寄存器配置映射 |
| `historical_data` | 历史采集数据 |
| `alarm_records` | 报警记录 |
| `alarm_rules` | 报警规则 |
| `system_logs` | 系统日志 |

### 8.3 性能优化

#### 大规模设备管理

- 100+设备时建议增加轮询间隔（2000ms以上）
- 使用线程池限制并发连接数
- 定期清理历史数据以减小数据库体积

#### 内存管理

- 系统支持100设备×200寄存器 = 20000总寄存器
- 内存占用约50-100MB（正常运行）
- 实时曲线图建议每个图表不超过4条曲线

## 常见问题

### Q1: 无法连接到 Modbus TCP 设备

**排查步骤**:
1. 使用 `ping <IP>` 测试网络连通性
2. 确认端口号（默认502）和从站ID
3. 检查防火墙是否阻止了502端口
4. 查看日志文件获取详细错误信息

### Q2: 串口设备无法识别

**Windows**:
1. 打开设备管理器 → 端口(COM和LPT)
2. 查看COM口号
3. 确认串口未被其他程序占用

**Linux**:
```bash
# 查看可用串口
ls /dev/ttyUSB* /dev/ttyACM*

# 添加用户到dialout组
sudo usermod -a -G dialout $USER
```

### Q3: 读取的数据不正确

**可能原因**:
1. 寄存器地址偏移（Modbus地址从0开始，某些设备文档从1开始）
2. 数据类型不匹配（Int16 vs Float）
3. 字节顺序问题（大端/小端）

**解决方法**:
1. 核对设备手册中的寄存器地址
2. 尝试不同的数据类型
3. 联系设备厂商确认字节顺序

### Q4: 界面卡顿

**解决方法**:
1. 在设置中增加轮询间隔
2. 减少同时监控的设备数量
3. 减少实时曲线图的数据点数
4. 关闭不必要的实时图表

### Q5: 如何备份数据

**备份内容**:
1. 配置文件 (`config.json`)
2. 数据库文件 (`equipment_management.db`)
3. 导出重要数据为 CSV/Excel

**恢复步骤**:
1. 将备份文件复制回配置目录
2. 重启程序
