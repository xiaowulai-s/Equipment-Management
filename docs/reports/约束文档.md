## 1️⃣ **项目架构与开发流程约束**

```
请遵循以下项目架构要求进行开发：

1. 创建一个标准的四层解耦架构：UI层、设备管理层、通信层、协议层。每个模块必须独立，且模块之间通过接口进行通信。
2. 代码必须遵循PEP 8规范，使用类型提示，异常必须记录日志，Git提交信息符合约定式提交规范。
3. 项目必须使用Python 3.8+，且必须提供完整的 `requirements.txt` 依赖管理文件。
4. 每个模块必须有详细的接口文档和单元测试，单元测试覆盖率不低于80%。
5. 代码中必须使用QSS样式进行UI美化，所有布局都必须使用Qt Layout（QVBoxLayout, QHBoxLayout 等），禁止使用 setGeometry 或 move。

```

***

## 2️⃣ **协议层与通信层实现约束**

```
在实现协议层时，必须遵循以下要求：

1. 协议基类（如Modbus协议基类）必须继承自 `QObject`，并且使用信号槽机制进行通信。
2. 所有协议（ModbusTCP、ModbusRTU、ModbusASCII）必须支持异步操作和超时机制。
3. 必须实现清晰的信号：`connected`、`disconnected`、`data_received`、`error_occurred`。
4. 在实现协议时，所有协议操作必须有统一的异常处理机制，且要记录日志。
5. 在通信驱动层，所有通信接口（open、close、read、write）必须线程安全，支持超时配置。
6. 使用适当的协议调试工具（例如Wireshark）来确保协议正确实现，并输出必要的调试日志。

```

***

## 3️⃣ **设备管理与数据层实现约束**

```
设备管理层的实现必须符合以下要求：

1. 设备模型必须包含设备ID、名称、协议类型、通信参数等属性，并支持JSON序列化。
2. 设备状态（如：DISCONNECTED、CONNECTED、ERROR）必须通过信号机制进行通知，必须支持异步操作。
3. 设备管理器必须实现设备的增、删、改、查功能，并支持设备数据缓存。
4. 必须支持设备批量操作，并能够处理设备异常，确保在出现异常时及时通知用户。
5. 数据存储使用SQLite数据库，数据库操作必须实现事务处理，确保数据一致性。
6. 数据仓库必须支持CRUD操作，并能够处理大规模数据，查询和导出功能必须支持分页。
7. 必须编写单元测试，确保每个功能模块的正确性和性能。

```

***

## 4️⃣ **UI开发约束**

```
UI开发必须遵循以下约束：

1. 所有UI必须使用Qt布局系统，不允许使用 `setGeometry` 或 `move` 进行绝对布局。
2. 所有UI组件必须有清晰的状态指示，必须使用QSS进行样式定制。
3. 必须实现界面自适应功能，确保在窗口大小变化时UI布局不被破坏。
4. 必须按照功能模块进行UI划分，如：设备管理UI、寄存器配置UI、数据可视化UI等。
5. 所有UI界面必须具备响应式布局，支持大屏和小屏设备。
6. 必须使用PySide6中的信号槽机制进行UI与后台的交互，所有UI更新必须通过信号通知进行。
7. 所有UI必须通过单元测试进行验证，确保交互流程无误，UI表现符合预期。

```

***

## 5️⃣ **数据可视化与报警系统约束**

```
在数据可视化和报警系统的实现中，必须遵循以下要求：

1. 数据可视化部分（如实时曲线图、仪表盘、趋势图）必须使用pyqtgraph或matplotlib实现，所有图表必须支持动态更新和缩放。
2. 必须实现报警规则配置功能，支持高阈值、低阈值等多种报警规则。
3. 报警系统需要支持多级报警（INFO、WARNING、ERROR、CRITICAL），并能够通过界面实时显示报警信息。
4. 报警通知必须支持多种方式，如声音、弹窗、邮件等。
5. 所有报警触发和清除操作必须记录日志，并通过信号通知相关模块。
6. 数据可视化部分必须提供导出功能，支持将图表数据导出为CSV、Excel等格式。

```

## 6️⃣ **测试与优化约束**

```
测试和优化要求：

1. 所有功能模块必须编写单元测试，覆盖率不低于80%。测试必须覆盖边界条件、异常处理、性能等方面。
2. 集成测试必须对系统的关键功能进行端到端测试，确保模块间的兼容性。
3. 在实现过程中，必须使用性能分析工具（如cProfile或Py-Spy）优化代码性能，确保系统能够处理大规模设备数据。
4. 必须对数据库查询、UI渲染、数据采集等关键流程进行性能优化，避免性能瓶颈。
5. 优化系统启动时间，确保设备连接、数据加载、UI渲染都能在合理时间内完成。
6. 测试过程中必须记录详细的测试报告，并进行性能对比分析。

```

```
在项目开发的过程中，必须遵循以下CI/CD要求：

1. 配置CI/CD管道，自动化构建、测试和部署。
2. 每次提交代码时，必须运行单元测试，确保无测试失败。
3. 代码审查时，必须确保代码符合PEP 8规范，且具备必要的文档注释和异常处理。
4. 必须使用SonarQube等工具进行代码质量分析，确保每个模块的代码质量达到标准。
5. 项目必须支持Docker化，确保能够在不同环境中顺利部署。
6. 自动化部署时，必须支持版本管理，确保每次发布的版本都能追溯。

```
